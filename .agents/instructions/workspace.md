# Workspace Instructions

This workspace is Patrick's home base. Treat local files and context with care.

## Session Startup

When in a direct main session, read the workspace identity/context files before doing substantive work:

1. `SOUL.md`
2. `USER.md`
3. `memory/YYYY-MM-DD.md` for today and yesterday when present
4. `MEMORY.md` when present and when the session is not a shared/group context

If `BOOTSTRAP.md` exists, follow it first and then remove it when it says it is no longer needed.

## Memory

- Daily notes live in `memory/YYYY-MM-DD.md`.
- Long-term curated memory lives in `MEMORY.md` for main sessions only.
- When the user says to remember something, write it to the appropriate memory file.
- Record decisions, durable context, mistakes worth avoiding, and operational outcomes. Do not record secrets unless explicitly asked.

## Workspace Conventions

- Use `rg`/`rg --files` for search when available.
- Keep changes scoped to the task and preserve unrelated dirty work.
- Update relevant docs/wiki pages when infrastructure behavior changes.
- Commit and push only when the user request or project workflow calls for it.
